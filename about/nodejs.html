NodeJS: The Browser and the Server Had Never Been This Close

NodeJS can be compared to one of those cool parents: It lets the young love thrive, doesn’t have an attitude when the lovers —the browser and the server in this case— want to have a movie date. In fact, it supports the relationship by providing an unwavering connection from the browser back to the server. And it uses minimal system resources while assuming this role. What this all means is that if coders use NodeJS to write an application, updates are sent to users straightaway in real time.

Where There is Constant Communication, There is NodeJS

Then, when do we prefer to use this kind of a framework? The answer is anytime we know the system architecture requires a concurrent dialogue between the browser and the server. What comes to mind first when we talk about a concurrent dialogue is, of course, multi-user, real-time applications such as online games, chat rooms, collaboration tools, and the like. When one user does something on the application that other user needs to see immediately without refreshing the page —and we all know how annoying having to refresh the page frequently could be— it is best to knock on Node’s door. Do we need to say that a framework with such characteristics compiles and executes data as fast as the Road Runner runs away from Coyote? Some might say, “But wait! Ruby, Django and Python could do this too… So why should I use NodeJS instead of one of these big boys?” We would tell them back, that NodeJS does the job without putting so many burdens on the server.

Open-Source Project Backed by a Real Company: NodeJS

We coders are aware that open source projects carry the risk of being abandoned by individual volunteers at one point in time by nature. Exceptions are possible if such open source projects are backed by real companies, and guess what? NodeJS is one of them! Presently, Joyent assumed the responsibility of sponsorship. There are other giants backing the project enthusiastically such as Wal-Mart, Microsoft, Yahoo, and PayPal.
